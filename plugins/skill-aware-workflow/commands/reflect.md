# タスク振り返り

**skill-proposer** スキルを使用して、直前のタスク実行を振り返る。

## 振り返り対象

$ARGUMENTS が指定されていれば、そのタスクについて振り返る。
指定がなければ、直前に完了したタスクを対象とする。

## 2種類のスキル提案

### 1. テクニカルスキル

実行中の摩擦から学んだ技術的なノウハウ:
- ライブラリ選択の知見
- 設定手順
- エラー回避方法

### 2. ワークフロースキル

複数ステップのプロセス全体:
- 事業計画 → PRD → 実装
- 調査 → 設計 → 実装 → テスト
- 要件定義 → デザイン → 開発

## 分析項目

### テクニカルスキル向け

- 明示的な修正指示（「〇〇に変えて」「それは違う」）
- 繰り返し失敗したツール/ライブラリ
- アプローチの変更（ピボット）
- 最終的にうまくいった方法

### ワークフロースキル向け

- 実行したフェーズの数と内容
- 各フェーズでの主要な成果物
- フェーズ間の依存関係
- 再利用可能なプロセスパターン

## 出力形式

```markdown
## 📊 タスク振り返り

### タスク概要
[タスクの要約]

### 実行フロー
1. [フェーズ1]: [内容]
2. [フェーズ2]: [内容]
3. ...

### 摩擦ポイント（あれば）
- 🔴 [摩擦の説明]

### 学び
- ✅ [学び1]
- ✅ [学び2]

---

## 💡 スキル作成提案

### ワークフロースキル（該当する場合）

**提案: `[workflow-name]`**

検出されたフロー:
1. [フェーズ1] → 2. [フェーズ2] → 3. [フェーズ3]

このワークフローをスキル化すると:
- 次回同様のプロジェクトでガイドとして活用
- チェックリストとして使用可能

### テクニカルスキル（該当する場合）

**提案: `[skill-name]`**

学んだこと:
- [技術的な知見]

このスキルを作成すると:
- 同じ摩擦を回避できる

---

**どのスキルを作成しますか？**
1. 両方作成
2. ワークフローのみ
3. テクニカルのみ
4. 今回は作成しない
```

## スキル作成への引き継ぎ

ユーザーが作成を選択した場合:

1. 提案内容を **skill-creator** スキルに引き継ぐ
2. ワークフローかテクニカルかを明示
3. 今回のタスクを具体例として活用
4. スキル作成プロセスを開始

## 提案を控えるケース

- 単純なタスク（1-2ステップ）
- 摩擦もワークフローもない
- 既存スキルで対応可能
